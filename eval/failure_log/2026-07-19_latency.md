# 2026-07-19 — Legacy CRM latency exceeded the decision budget

## What broke

Before hardening, an injected six-second CRM delay merely made the entire graph slow. Retrieval eventually returned valid-looking data, so the agent could still approve a refund even though the configured four-second integration budget had already been violated. Operationally, this hid a dependency incident and made request latency unbounded.

## Diagnosis

The database driver's connection timeout covered connection establishment, not artificial delay before the query. No elapsed-time guard converted the injected fault into evidence loss. The trace therefore showed a slow success instead of a timeout.

## What changed

The shared fault boundary now receives the integration's latency budget. When injected latency reaches or exceeds that budget, it waits only through the budget and raises a typed `IntegrationUnavailable` timeout. The retrieval node records the system and error kind, increments an integration-error metric, and the policy gate returns low-confidence human escalation. No payment action is called.

## Remaining limitation

This harness uses a synchronous client and a deterministic injection layer. A larger deployment should use async deadlines propagated across services, bounded retries with jitter, and a circuit breaker.
